# Mentora Database Schema

## Collections

### Users Collection

Stores all user information (students and faculty).

```json
{
  "_id": ObjectId("507f1f77bcf86cd799439011"),
  "email": "user@example.com",
  "name": "User Name",
  "password_hash": "pbkdf2:sha256:600000$...",
  "role": "student|faculty",
  "department": "Computer Science",
  "expertise": ["Python", "Machine Learning"],
  "bio": "User biography",
  "profile_picture": "https://example.com/image.jpg",
  "created_at": ISODate("2024-01-15T10:30:00.000Z"),
  "updated_at": ISODate("2024-01-15T10:30:00.000Z"),
  "is_active": true
}
```

**Indexes:**
- Unique index on `email`

**Field Descriptions:**
- `_id`: MongoDB ObjectId (auto-generated)
- `email`: User's email address (unique)
- `name`: Full name
- `password_hash`: Hashed password
- `role`: Either "student" or "faculty"
- `department`: Academic department
- `expertise`: Array of expertise areas (for faculty)
- `bio`: User biography/description
- `profile_picture`: URL to profile image
- `created_at`: Account creation timestamp
- `updated_at`: Last profile update timestamp
- `is_active`: Account status

### Messages Collection

Stores all messages between users.

```json
{
  "_id": ObjectId("507f1f77bcf86cd799439012"),
  "sender_id": ObjectId("507f1f77bcf86cd799439011"),
  "receiver_id": ObjectId("507f1f77bcf86cd799439013"),
  "content": "Hello! I need help with...",
  "created_at": ISODate("2024-01-15T10:35:00.000Z"),
  "read": false
}
```

**Indexes:**
- Index on `[sender_id, receiver_id]`
- Index on `created_at`

**Field Descriptions:**
- `_id`: MongoDB ObjectId (auto-generated)
- `sender_id`: ObjectId of message sender
- `receiver_id`: ObjectId of message recipient
- `content`: Message text content
- `created_at`: Message timestamp
- `read`: Boolean indicating if message was read

---

### Connections Collection

Stores mentorship relationships between students and faculty.

```json
{
  "_id": ObjectId("507f1f77bcf86cd799439014"),
  "student_id": ObjectId("507f1f77bcf86cd799439011"),
  "faculty_id": ObjectId("507f1f77bcf86cd799439013"),
  "created_at": ISODate("2024-01-15T10:40:00.000Z"),
  "status": "active"
}
```

**Indexes:**
- Unique compound index on `[student_id, faculty_id]`

**Field Descriptions:**
- `_id`: MongoDB ObjectId (auto-generated)
- `student_id`: ObjectId of student
- `faculty_id`: ObjectId of faculty
- `created_at`: Connection creation timestamp
- `status`: Connection status ("active", "inactive", "pending")

## Query Examples

### Get all faculty members in a department

```javascript
db.users.find({
  role: "faculty",
  department: "Computer Science"
})
```

### Get conversation between two users

```javascript
db.messages.find({
  $or: [
    { sender_id: ObjectId("..."), receiver_id: ObjectId("...") },
    { sender_id: ObjectId("..."), receiver_id: ObjectId("...") }
  ]
}).sort({ created_at: 1 })
```

### Get unread messages for a user

```javascript
db.messages.find({
  receiver_id: ObjectId("..."),
  read: false
})
```

### Get all students connected to a faculty

```javascript
db.connections.find({
  faculty_id: ObjectId("..."),
  status: "active"
})
```

### Get expertise of all faculty

```javascript
db.users.find(
  { role: "faculty" },
  { name: 1, expertise: 1, department: 1 }
)
```

## Data Validation Rules

### Users
- `email`: Required, unique, valid email format
- `name`: Required, string (2-100 characters)
- `password_hash`: Required, generated by Werkzeug
- `role`: Required, "student" or "faculty"
- `department`: Required, string
- `expertise`: Optional array for faculty
- `bio`: Optional, max 500 characters

### Messages
- `sender_id`: Required, must be valid user ObjectId
- `receiver_id`: Required, must be valid user ObjectId
- `content`: Required, non-empty string
- `created_at`: Auto-generated on insert
- `read`: Default false

### Connections
- `student_id`: Required, must reference a student user
- `faculty_id`: Required, must reference a faculty user
- `status`: Optional, default "active"
- One-to-one relationship enforced by unique index

## Aggregation Pipeline Examples

### Get message statistics for a user

```javascript
db.messages.aggregate([
  {
    $match: {
      $or: [
        { sender_id: ObjectId("...") },
        { receiver_id: ObjectId("...") }
      ]
    }
  },
  {
    $group: {
      _id: null,
      total_messages: { $sum: 1 },
      unread: {
        $sum: {
          $cond: [{ $eq: ["$read", false] }, 1, 0]
        }
      }
    }
  }
])
```

### Get faculty with most connections

```javascript
db.connections.aggregate([
  {
    $group: {
      _id: "$faculty_id",
      student_count: { $sum: 1 }
    }
  },
  {
    $sort: { student_count: -1 }
  },
  {
    $lookup: {
      from: "users",
      localField: "_id",
      foreignField: "_id",
      as: "faculty_info"
    }
  }
])
```

## Performance Considerations

1. **Message Queries**: Index on `[sender_id, receiver_id]` enables efficient conversation retrieval
2. **User Lookup**: Single field indexes on collections improve query speed
3. **Pagination**: Implement limit and skip for large result sets
4. **Caching**: Consider caching faculty lists and user profiles

## Backup and Restore

### Backup MongoDB

```bash
# Backup to file
mongodump --uri "mongodb://localhost:27017/mentorship" --out ./backup

# Restore from backup
mongorestore --uri "mongodb://localhost:27017/mentorship" ./backup/mentorship
```

### MongoDB Atlas Backup

- Atlas automatically backs up data every 6 hours
- Manual snapshots can be created
- Restore points available for 30 days

## Security Notes

- Never expose MongoDB connection strings
- Use environment variables for URI
- Enable authentication on MongoDB
- Use SSL/TLS for connections
- Regularly update MongoDB
- Monitor database logs
- Implement rate limiting for API queries
- Validate and sanitize all user input
